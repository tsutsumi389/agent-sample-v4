どんな評価基準を作ったか

機能の網羅性ではなく「美しさ・センス」を測るために、芸術的観点の8軸・計100点のルーブリックを設計しました。

評価思想 — 優れたUIは、機能の羅列ではなく一貫した美意識から生まれる「静かな佇まい」を持つ。ダークテーマのプロダクティビティツールにおける美しさとは、抑制された配色の中で情報が呼吸し、余白とリズムが視線を導き、細部の精度が信頼感を醸成することにある。本基準は実装の網羅性ではなく、色・字・間・律・個性という審美的要素が、いかに統御され、いかに感情を喚起するかを問う。装飾の多さではなく、引き算の精度と一貫性を高く評価する。

配色・色彩設計

18点

ダークテーマにおける階層的な背景色(bg/panel/elevated/hover)、アクセント色、状態色(danger/running/done)の選定と調和。明度差・彩度設計、コントラストの心地よさ、色数の抑制と意味付けの統一性を測る。

優 (90%+)

5段階の背景色が緻密な明度ステップで階層を形成し、単一のブルー系アクセントが規律正しく使われる。running(琥珀)/done(緑)/danger(赤)の状態色が背景に溶け込む半透明tintで上品に共存し、色が情報構造そのものを語る。どの色も偶然でなく意図を感じさせる。

劣

背景色の明度差が曖昧で階層が潰れる、またはアクセントや状態色が乱立して目が疲れる。彩度が高すぎ/低すぎてダークテーマの落ち着きを損なう。色の意味付けに一貫性がなく、場当たり的な色選定に見える。

タイポグラフィ

15点

フォント選定(日本語フォントスタック含む)、サイズスケール、行間、字間、太さ、本文と等幅(コード/ツール)の使い分け。可読性と表情の両立を測る。

優 (90%+)

日本語を見据えたHiragino/Noto系のフォントスタックが整備され、11/12/13/14/16/18pxの明快なスケールと1.6前後の快適な行間を持つ。タイトルの微細なletter-spacing、コード/ツール名への等幅フォント適用など、文字の役割分担が洗練され、画面全体に静かな統一感を与える。

劣

フォントサイズが無秩序で階層が読めない、行間が詰まりすぎ/空きすぎて読みづらい。日本語表示への配慮(フォントスタック・行間)が欠け、等幅と本文の使い分けが雑。文字組みに無頓着で安っぽい印象を与える。

余白と構図

15点

3カラムgridの比率(260/1fr/320)、各パネルの内外パディング、要素間ギャップの統一感。情報密度と呼吸感のバランス、レイアウトの安定した骨格を測る。

優 (90%+)

3カラムの幅比がコンテンツの主従を的確に表し、中央チャットが主役として広く呼吸する。8pxを基調とした余白・gapが全コンポーネントで一貫し、メッセージバブルのmax-width(76%)やパディングが心地よい密度を作る。窮屈さも間延びもない、計算され尽くした構図。

劣

カラム比が不自然でチャットが狭い/サイドバーが過大。パディングやgapの値がバラバラで余白に律動がない。要素が詰め込まれ窒息するか、逆に散漫に広がり焦点を失う。構図に骨格が感じられない。

視覚的階層とリズム

15点

text/text-dimによる主従の明示、見出し・メタ情報・本文の差別化、user/assistantバブルの非対称配置、繰り返し要素(スレッド項目・記憶カード)のリズム。視線誘導の自然さを測る。

優 (90%+)

主要テキストと減光テキスト(text-dim)の対比が情報の主従を即座に伝え、日付・スコア・ロールなどのメタ情報が控えめに脇を固める。userは右・assistantは左の非対称配置が会話のリズムを生み、リスト項目の反復が心地よい拍を刻む。視線が迷わず流れる。

劣

すべての文字が同じ強さで主従が読めない、メタ情報が本文と競合する。バブルの左右配置やリストの反復に律動がなく、視線が彷徨う。強調すべき箇所(送信ボタン等)が埋もれ、階層が平板。

ディテールとマイクロインタラクション

13点

hover/focus/active/disabled状態の表現、トランジション、カーソル点滅・進捗表示・ツールチップのアニメーション、角丸・境界線・pill形状などの仕上げの精度を測る。

優 (90%+)

hoverで現れる削除ボタン、focus時のアクセントoutline、selectで切り替わるツールチップ三角、blinkカーソル、棒グラフのwidthトランジションなど、細部の挙動が一貫したタイミング(0.15s前後)で上品に動く。角丸・pill・半透明tintの仕上げが精密で、触れる前から品質を予感させる。

劣

hover/focus/disabledの状態表現が欠落または唐突。トランジションが無い/長すぎ/不揃いで安っぽい。角丸や境界線の値が不統一、アニメーションが過剰または雑で、細部に手抜きが透ける。

一貫性とデザインシステム

12点

CSS変数によるトークン化、半径・色・余白・フォントの再利用、ボタンや入力欄など共通コンポーネントの統一。ばらつきの無さと体系の強度を測る。

優 (90%+)

色・半径・フォントがCSS変数として一元管理され、btn/input/cardが共通スタイルから派生する。新旧の機能(チャット/記憶/生成的UI/モーダル)を跨いでも同じ語彙で語られ、どの画面を見ても単一の作者の手を感じる。デザインシステムとして自立している。

劣

ハードコードされた色や半径が散在し、同種要素ごとに値が微妙に異なる。後付け機能だけ作法が違い、つぎはぎに見える。共通化の意図が読めず、全体が一つの作品として成立していない。

世界観と個性

7点

このUI固有の雰囲気・トーン(ローカルAIエージェントらしい知的で落ち着いた佇まい)、記憶パネルやツールコール表現などの独自モチーフが醸す個性を測る。

優 (90%+)

抑制されたダークトーンと知的なブルーのアクセントが「思考するエージェント」の世界観を静かに体現する。記憶カード・ツールチップ・生成的UIといった独自要素が、汎用チャットを超えた固有の人格を画面に与え、記憶に残る佇まいを持つ。

劣

どこにでもある汎用ダークチャットの域を出ず、固有の雰囲気が無い。独自機能が視覚的個性に昇華されず、テーマやトーンに作り手の世界観が感じられない。

美観としてのアクセシビリティ

5点

審美性を損なわない範囲でのコントラスト確保、focus可視性、状態色の判別性、減光テキストの可読限界など、美しさと配慮の両立を測る。

優 (90%+)

本文テキストと背景のコントラストが快適な可読域にあり、focus outlineが美観を保ったまま明示される。text-dimが薄すぎず情報を保ち、状態色が色覚に頼りすぎずラベルと併存する。配慮が美しさと矛盾せず統合されている。

劣

減光テキストや境界線が薄すぎて判読困難、focusが見えず操作の所在を失う。状態を色だけに依存し判別しづらい。美観のためにアクセシビリティが犠牲になっている。

配色・色彩設計13/18【色彩・タイポグラフィ】--bg #0f1117 → --bg-panel #161922 → --bg-elevated #1e2230 → --bg-hover #252a3a と4段階の背景が緻密かつ単調増加する明度ステップで階層を形成し、--border #2a2f40 がその延長線上に綺麗に乗っている点は規律正しい。アクセントは --accent #5b8def 単一のブルー系に統一され、知的で落ち着いたダークトーンに馴染む。状態色も tool-chip-status.running が rgba(229,192,123,0.15) の琥珀tint、.done が rgba(152,195,121,0.15) の緑tint、エラーが rgba(224,108,117,0.1) の赤tintと、いずれも半透明tint+同系文字色で上品に背景へ溶け込み「色が情報構造を語る」域に近い。減点理由は3つ。(1) 状態色のtint文字色 #e5c07b/#98c379/--danger #e06c75 がOne Dark由来のパレットで、--accent の #5b8def とは別系統の色温度・彩度感を持ち、厳密には『偶然でない単一作者の調色』とまでは言い切れない。(2) --user-bubble #2d3a5c が背景階層ともアクセントとも独立した第6の色として存在し、bg系の明度ステップから外れる。(3) chart-fill が --accent ベタ塗りで、データ可視化に独自の色設計がない。展示会水準の『どの色も意図を感じさせる』には半歩届かない。

余白と構図12/15【色彩・タイポグラフィ】grid-template-columns:260px 1fr 320px は中央チャットを主役に据え、サイドバーを絞り右の記憶パネルをやや広めに取る主従が的確。message-list の padding:24px 32px、composer の 16px 32px 20px と左右32pxで軸が揃い、message-bubble max-width:76% は会話の呼吸として心地よい密度。8px基調(gap:6/8/10/12)もおおむね一貫。減点は、余白値が8の倍数で完全には統一されておらず 6/10/14/22px が混在(thread-item padding:8px 10px、btn padding:6px 14px、markdown ul padding-left:22px 等)し、厳密なスペーシングスケールではない点。また message-row margin-bottom:16px に対しバブル内 padding:10px 14px とやや縦が詰まり気味で、長文Markdown時のリズムは中央32pxパディングに対してバブルが76%まで広がると行長が長くなりすぎる懸念。骨格は明確だが律動の精度は展示会水準にわずかに届かない。
視覚的階層とリズム11/15【色彩・タイポグラフィ】--text #e2e6f0 と --text-dim #8b91a7 の対比が主従を即座に伝え、message-role 11px/dim、thread-date 11px/dim、memory-meta 11px/dim、memory-score をアクセント色にするなどメタ情報の控えめな脇固めは的確。message-row.user の justify-content:flex-end による右寄せと assistant 左寄せの非対称配置が会話のリズムを生み、thread-item / memory-item の反復も拍を刻む。送信ボタン btn.send はアクセント塗り+weight600で明確に浮く。減点理由: (1) user バブルと assistant バブルがどちらも『あなた/アシスタント』の message-role ラベルを持つため、非対称配置の効果が冗長なラベルでやや薄まる(右寄せなら色や形だけで話者は自明)。(2) memory-item と modal-item と ui-resource がいずれも『bg-elevated+border+radius8』のほぼ同一カードで、繰り返し要素間の視覚的役割差が乏しく、画面によっては反復が単調。(3) tool-chip の running/done が同サイズ同形で、進行と完了のリズム差が色だけに依存。階層は読めるが視線誘導の妙味は中程度。
